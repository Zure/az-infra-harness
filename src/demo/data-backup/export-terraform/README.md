# Customer Portal — Terraform Infrastructure

Generated by Azure Infrastructure Prompt Kit.

## Prerequisites

- Terraform CLI (>= 1.5)
- Azure CLI (`az`)
- Azure subscription with Contributor access
- Service principal or managed identity for CI/CD

## Quick Start

```bash
# Login to Azure
az login

# Initialize Terraform
cd export-terraform
terraform init

# Deploy to dev
terraform plan -var-file=environments/dev.tfvars
terraform apply -var-file=environments/dev.tfvars
```

## Remote State Setup

1. Create a storage account for Terraform state
2. Uncomment and configure `backend.tf`
3. Run `terraform init` to migrate state

## Environments

| Environment | Variable File | Description |
|-------------|--------------|-------------|
| dev | `environments/dev.tfvars` | Development — minimal SKUs, scale-to-zero |
| staging | `environments/staging.tfvars` | Staging — mid-tier SKUs, zone redundancy |
| prod | `environments/prod.tfvars` | Production — full SKUs, geo-redundancy |

## CI/CD

The GitHub Actions workflow at `.github/workflows/deploy-infra.yml` provides:

- **Validate**: Format check, init, validate
- **Plan**: Terraform plan for dev
- **Deploy Dev**: Automatic on push to `develop`
- **Deploy Staging**: Automatic on push to `staging`
- **Deploy Prod**: Requires manual approval, on push to `main`

Configure these repository secrets:
- `AZURE_CLIENT_ID`
- `AZURE_TENANT_ID`
- `AZURE_SUBSCRIPTION_ID`
