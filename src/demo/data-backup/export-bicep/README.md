# Customer Portal — Bicep Infrastructure

Generated by Azure Infrastructure Prompt Kit.

## Prerequisites

- Azure CLI (`az`) with Bicep extension
- Azure subscription with Contributor access
- Service principal or managed identity for CI/CD

## Quick Start

```bash
# Login to Azure
az login

# Deploy to dev
az deployment sub create \
  --location westeurope \
  --template-file main.bicep \
  --parameters parameters/main.dev.bicepparam
```

## Environments

| Environment | Parameter File | Description |
|-------------|---------------|-------------|
| dev | `parameters/main.dev.bicepparam` | Development — minimal SKUs |
| staging | `parameters/main.staging.bicepparam` | Staging — mid-tier SKUs |
| prod | `parameters/main.prod.bicepparam` | Production — full redundancy |

## CI/CD

The GitHub Actions workflow at `.github/workflows/deploy-infra.yml` provides:

- **Validate**: Bicep lint and what-if preview
- **Deploy Dev**: Automatic on push to `develop`
- **Deploy Staging**: Automatic on push to `staging`
- **Deploy Prod**: Requires manual approval, on push to `main`

Configure these repository secrets:
- `AZURE_CLIENT_ID`
- `AZURE_TENANT_ID`
- `AZURE_SUBSCRIPTION_ID`
