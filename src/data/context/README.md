# Context Section

This directory contains the context definition files that describe the deployment landscape for your application.

## Files

### Markdown Files (Planning Boxes)

Generated by coding agents when users run the specified commands:

- `infrastructure-context.md` - Infrastructure landscape (VNets, landing zones, subscriptions)
  - Command: `/infrastructure-context`
  - Questions: Landing zones, network topology, existing resources, connectivity

- `platform-context.md` - Platform services (monitoring, security, identity)
  - Command: `/platform-context`
  - Questions: Azure Monitor, Key Vault, identity services, platform policies

- `development-context.md` - Development tooling (CI/CD, deployment processes)
  - Command: `/development-context`
  - Questions: CI/CD pipelines, deployment tools, development practices

## How It Works

1. User sees empty planning boxes with commands
2. User copies command (e.g., `/infrastructure-context`)
3. User runs command in coding agent
4. Coding agent asks relevant questions
5. Coding agent generates markdown file in this directory
6. User refreshes browser - content appears in planning box
7. Box shows checkmark and blue border when complete

## Data Format

All files are markdown with structured headings. The content is displayed using react-markdown with the typography plugin for proper formatting.

## Completion Tracking

A box is considered complete when its markdown file exists and contains content. The Context component tracks completion and enables the "Continue" button when all three boxes are filled.
