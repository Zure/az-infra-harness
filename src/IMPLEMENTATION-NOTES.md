## How It Works

### For Developers Using Coding Agents

1. **Navigate to Application Definition section** in the app
2. **Copy the command** from an empty planning box (e.g., `/application-overview`)
3. **Paste into your coding agent** (like OpenCode)
4. **The coding agent generates markdown content** and saves it to:
   - `src/data/application-definition/application-overview.md`
   - `src/data/application-definition/non-functional-requirements.md`
   - `src/data/application-definition/components/*.json`
5. **Refresh the page** to see the updated content
6. **Click "Continue to Context"** when all three boxes are complete

### Data File Locations

The coding agent should write files to these locations:

```
src/data/application-definition/
├── README.md                              # Documentation
├── application-overview.md                # Generated by /application-overview
├── non-functional-requirements.md         # Generated by /non-functional-requirements
└── components/                            # Generated by /application-components
    ├── web-frontend.json
    ├── api-backend.json
    └── ...
```

### Component JSON Format

```json
{
  "id": "unique-id",
  "name": "Display Name",
  "type": "compute|data|networking",
  "description": "Component description"
}
```

## Architecture

- **Server Components**: Page components that load data server-side
- **Client Components**: Interactive components with event handlers
- **Props-based Design**: All UI components receive data via props
- **File-based Data**: Simple, git-friendly markdown and JSON files
- **Type-safe**: Full TypeScript support with comprehensive types
